

You are working on an AI-driven Islamic application called "Tawbah".

IMPORTANT ARCHITECTURE RULE:
Do NOT treat pages as independent features.
All existing routes are capabilities that must be orchestrated by a central AI system called "Zakiy".

Your task is to refactor and extend the system into an AI-Orchestrated Architecture.

## Core Concept

Zakiy is the central decision engine that:

* Understands user state
* Detects risk
* Assigns tasks
* Redirects user to the correct capability (route)

## Step 1: Create Core AI Service

Create a service:
`/src/core/zakiy-engine.ts`

It should:

* Accept user context
* Return:

  * message
  * action (route redirect)
  * task

## Step 2: Create User State System

Create:
`/src/core/user-state.ts`

State includes:

* level
* streak
* riskScore
* lastRelapse
* dangerTimes

## Step 3: Create Risk Engine

Move logic from:

* danger-times
* sos

Into:
`/src/core/risk-engine.ts`

Expose:

* analyzeRisk()
* shouldTriggerSOS()

## Step 4: Create Task Engine

Merge:

* hadi-tasks
* habits
* plan

Into:
`/src/core/task-engine.ts`

## Step 5: Refactor Home Page

Home should NOT show many options.

It should:

* Show current state
* Show ONE primary button:
  "Start Now"

On click:
→ call Zakiy
→ follow returned action

## Step 6: Create Central API

Create endpoint:
POST /api/zakiy

Returns:
{
message,
action,
task
}

## Step 7: Community Refactor

Merge:

* ameen
* secret-dua
* dhikr-rooms
* challenge
* map

Into:
Single module:
`/community`

## Step 8: Notification System

Convert to event-driven:

* risk spike
* inactivity
* prayer time

## FINAL RULE

Zakiy is the ONLY decision maker.
Pages are just execution tools.

Start with step 1 and implement incrementally.
